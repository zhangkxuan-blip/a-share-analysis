# 输出回归经验与防复发规则

## 适用场景

每次用真实 A 股生成完整报告后，用本文件复盘。目标是减少“信息很多但难决策”和“数字精确但证据不够”的问题。

## 已验证的常见失败模式

| 失败模式 | 风险 | 防复发规则 | 验证方式 |
| --- | --- | --- | --- |
| 开头与结尾行情价格不同 | PE、估值空间和价位互相冲突 | 双源核验后冻结 snapshot ID，所有价格依赖项整体重算 | 搜索报告中的 snapshot ID 和当前价，仅允许一组 |
| 18 项后又按六模块重复 | 小白难以抓住结论 | 默认三层输出，18 项只出现一次，仅展开 3–5 个决定因素 | 检查是否存在逐项重复段落 |
| 给出 /100 分但不显示缺数 | 分数产生虚假确定性 | 显示 earned/available、覆盖率和置信度；覆盖低于 70% 不归一化 | 评分卡必须含四列 |
| 不同供应商 PE/PB 历史直接算百分位 | 分母和股份口径不可比 | 先过 same-definition gate；失败只给方向性迹象 | 检查口径清单 |
| 主观情景概率自动加权 | 伪造一个貌似精确的目标价 | 未校准概率不加权，保留三情景区间 | 测试 `probabilitiesCalibrated` 默认 false |
| PE、PB、技术价位被当成三个“独立”锚 | 底部区间过度自信 | 按锚点家族去重，至少两个家族且需要接近 | 测试同家族和高离散样本均 unavailable |
| 用全天均量线性折算盘中量比 | 忽视 A 股日内 U 形成交 | 优先同一时刻五日均量；线性法标记低置信代理 | 输出必须带 benchmark mode |
| 用总股本算换手却称自由流通换手 | 分母名称错误 | 输出 denominatorClass 和日期 | 只允许 free_float/tradable/total |
| 将涨跌量或大/小单写成资金、机构、散户流入 | 把交易特征误当身份和资本流 | 分开 trade-size proxy 与 price-volume pressure，禁止身份结论 | 检查 identity/capitalFlow 许可字段均 false |
| 用不同日期媒体摘要拼板块 5/20 日排名 | 窗口不可比 | 同分类、同窗口、同指标；失败 unavailable | 板块表需记录窗口与分类体系 |
| 用少量论坛帖子打情绪分 | 样本偏差被放大 | 记录样本卡并执行 10/20 条门槛与去重 | 评分前检查平台、窗口、N 值 |
| 将新中标与在手订单相加 | 订单重叠导致虚增 | 未确认非重叠只分开展示公告总额与净新增已知 | 检查公司是否给出桥接表 |
| 用“技术国产”推导“采购不依赖进口” | 混淆技术、采购与收入地域 | 建立组件级进口依赖矩阵，缺供应商地域就写 unknown | 每个关键部件单列证据 |
| 把研发投入等同明星产品商业成功 | 忽略量产和收入确认 | 分阶段展示公告、验证、出货、量产和收入 | 产品结论必须带商业化阶段 |
| 只用最新定期报告计算 TTM，漏掉之后的业绩预告或快报 | 盈利已剧变时估值仍停留在旧周期，PE 和情景价会严重失真 | 定期报告后再扫描业绩预告、业绩快报和新入库报告；实际数与预告分层，并用更新披露建立第二估值锚 | 财务截止卡必须列出“最新报告”和“报告后披露”，估值说明采用哪一层 |
| 把收盘后披露的财报当作已经反映在收盘价中 | 催化剂的定价状态被倒置，买点判断产生前视偏差 | 比较公告实际发布时间与冻结行情时间；晚于快照则标记 not_reflected_in_snapshot_price | 事件卡必须显示行情快照时间、公告时间和是否已进入可交易信息集 |
| 多股票报告直接给精确“优先研究顺序”但不展示评分 | 小白容易把研究顺序理解为买入推荐，缺失数据也被隐藏 | 默认使用公司质量与估值/市场双轴分层；线性顺序须展示得分、可得分、覆盖率、缺失处理和 tie-breaker | 批量报告开头不得出现无评分依据的 1-N 总排名 |
| 三情景估值只列三个价格 | 无法判断盈利口径、倍数和一次性损益，形成伪精确 | 每股价值必须附方法、核心口径、关键假设、截止、来源类别和置信度；前提不足则 unavailable 或低置信度交叉检查 | 检查每只股票情景行是否存在 method、basis、as-of 和 confidence |
| 只有均线、前低或价格位置却称为底部参考区 | 单一技术家族被包装成基本面底部 | 仅称技术支撑观察区并给失效条件；底部参考区仍需两个独立锚点家族重合 | 批量表的 bottom label 必须与 anchor-family gate 一致 |
| 精确机构持仓比例没有截止和分母 | 陈旧披露或总股本/流通股口径被误读 | 每个比例同时显示截止、分母、机构类型和披露滞后；缺任一关键字段则不输出精确占比 | 机构单元格必须含 as-of 和 denominator |
| 只用板块收益率就描述主要资金流入和未来领涨 | 相对收益、成交拥挤和真实资金身份被混为一谈 | 缺宽度、成交额占比及独立 ETF/融资证据时，只能写相对收益领先，完整资金流向与领涨排名 unavailable | 板块表检查 comparability gate 和 flow evidence |
| 批量报告沿用单股长文，18 项反而部分消失 | 篇幅很长但覆盖不可追踪，重要缺口被叙述淹没 | 使用三张跨股表覆盖 18 项，每个证券都能定位或明确 unavailable | 静态检查批量契约中的 1–18 映射和三表结构 |
| 关键盘后新披露在数据处理中识别了但最终报告遗漏 | 用户无法知道下一交易日存在未定价事件 | 在第一层单列盘后事件卡，禁止被篇幅压缩规则删除 | 检查 not_reflected_in_snapshot_price 事件数与事件卡数量一致 |
| 原始行情接口成为唯一用户可点击证据 | 接口可能需要请求头、返回难读或随后失效 | 原始接口留在证据台账，并补可访问行情页或官方收盘来源 | 关键行情至少有一个可访问展示来源 |

## 每次真实报告后的回归步骤

1. 检查唯一行情快照及所有派生项。
2. 检查评分覆盖率和所有 unavailable 项。
3. 抽查估值口径、订单重叠、进口依赖与资金措辞。
4. 统计 18 项是否各出现一次且无重复长文。
5. 运行全部测试，并把新失败模式补到本表和对应规则/测试。
