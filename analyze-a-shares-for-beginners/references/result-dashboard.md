# 结果优先输出面板

## Default Detail Level

默认使用三层输出，先决策、后证据。除表格外避免重复叙述；只展开会改变结论的 3–5 个因素。计算代入、长证据链和完整公式放在可选附录。

置信度：

- high：主要来源且有独立核验，定义和时点适配；
- medium：主要来源成立，但验证、时点或口径有一项不完整；
- low：间接代理、稀疏样本、主观情景或陈旧披露；
- unavailable：最低证据、可比口径或时间窗口不满足。

主张类型必须标注为 fact、derived metric、proxy 或 scenario。

两只及以上股票必须改用 [多股票批量比较输出契约](batch-comparison-output.md)。批量模式仍保持三层，但用跨股三表覆盖 18 项，禁止把单股长报告机械复制多份。

## 第一层：一屏决策简报

按以下顺序输出：

1. 一句话状态：公司质量、估值位置、市场状态三者分开说；
2. 冻结行情卡：代码、名称、价格、涨跌、成交额、行情时间、来源、snapshot ID；
3. 双评分卡；
4. 三个核心正向因素与三个核心风险；
5. 适合长线研究、事件波段、高风险短线、混合或当前不适合；
6. 当前状态、确认条件、转弱条件、失效条件、下一催化剂。

## 第二层：18 项紧凑决策表

使用列：编号、结论、决定性指标、截至时间、置信度、类型。每项一行，缺失就写 unavailable，不得删除。

1. 国家规划及核心政策相关性；
2. 历史估值位置及合理性；
3. 保守、中性、乐观每股估值；
4. 行业壁垒；
5. 周期阶段、预计持续时间及历史模式；
6. 龙头性、可替代性、稀缺性及溢价；
7. 短线或长线适配；
8. 国产替代及中美博弈相关性；
9. 进口、出口及国外供应依赖；
10. 盈利能力与现金质量；
11. 已披露机构名称及持仓比例；
12. 一周及一月股东户数代理趋势；
13. 5 日及 20 日资金/交易规模代理及可观测性；
14. 底部参考区间及置信度；
15. 周期、成长或混合属性及行业天花板；
16. 大盘 5 日/20 日领涨板块与未来 5 日/20 日情景；
17. 最新产品、研发投入、商业化阶段及预期；
18. 换手率与量比。

默认不再把这 18 项按六个模块逐段复述。仅在表后解释最关键的 3–5 项。

## 第三层：证据与缺口台账

| 结论模块 | 所需数据 | 实际可观测数据 | 使用方式 | 截至时间/陈旧度 | 缺口及补全方式 |
| --- | --- | --- | --- | --- | --- |

“使用方式”只能是直接证据、派生指标、明确代理或 unavailable。每个模块单独列数据截止时间，不能用报告总时间掩盖陈旧的持仓、股东户数或行业数据。

## 关键硬门槛

### 行情冻结

双源校验后冻结一个最终快照。所有 PE/PB、情景涨跌空间、技术价位、换手率和量比必须引用同一 snapshot ID。后续抓到新价格时，要么整体重算并生成新 snapshot ID，要么忽略新价，禁止混用。

### 历史估值

只有分母期间、股份口径、复权口径、合并范围和供应商算法可比时，才输出精确历史百分位。否则写“接近历史低位/高位的迹象（低置信度）”，并说明不可比项。

### 三情景估值

| --- | ---: | ---: | --- | --- | --- |
| 保守 |  |  |  |  |  |
| 中性 |  |  |  |  |  |
| 乐观 |  |  |  |  |  |

主观概率默认不生成“概率加权目标价”。只有概率来自明确历史基准、已校准模型，或用户明确要求并接受假设时才展示。

### 底部参考区

锚点按家族分组：技术结构、盈利/现金流估值、资产价值、交易/量能。至少两个不同家族进入相近区间才可形成底部参考区；只有同一家族多个指标，或锚点明显不重合时，写 unavailable 或“低置信度支撑观察区”。必须给失效条件。

### 资金、持仓与成交

- 已披露大股东/基金持仓可写机构名称和比例；
- 大单/小单仅叫 trade-size proxy，不能叫机构/散户流入流出；
- 涨跌日成交量、量价背离仅叫 price-volume pressure，不能叫资金流入流出；
- 换手率必须标注分母：free_float、tradable 或 total；
- 量比优先用过去五日同一交易时刻累计成交量；只有全天量时，标注 linear_clock_proxy 和低/中置信度，不能当成精确量比。

### 板块、情绪、订单与进口依赖

- 板块 5/20 日结论须采用同一分类体系、同一窗口和可比的收益/宽度/成交额占比；不满足则 unavailable；
- 情绪评分需说明平台、窗口、去重后样本量和互动权重。默认少于 10 条唯一样本为 unavailable；10–19 条只能低置信度定性；达到 20 条且至少两类独立来源后才可进入评分；
- 订单存量、新签合同、框架协议和中标公告可能重叠，未证明不重叠不得相加；分别展示“公告总额”和“已知净新增”；
- 技术自主、供应商国别、采购进口占比、收入出口占比分开写。公司没有披露供应商地域或进口份额时必须写 unknown，不得用“国产技术”推导“零进口依赖”。

## 结尾

列出 3–6 个后续监测指标、下一披露/催化时间、最重要的数据缺口和中性教育免责声明。可询问用户是否需要详细计算附录，但不要主动展开。

批量报告不得用未展示评分依据的精确总排名开头。先给公司质量轴和估值/市场轴，再给研究状态分层；只有透明评分覆盖达到 70% 且用户确实要求时，才增加有 tie-breaker 的线性顺序。
